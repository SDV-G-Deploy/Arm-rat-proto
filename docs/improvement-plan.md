# Improvement plan (итерация 1)

## P1 (делаем сейчас)
1. **Jump buffer (mobile-friendly):** окно ~120мс на принятие раннего прыжка.
2. **Input robustness на mobile-кнопке:** обработка `mouseleave`/`touchcancel`, чтобы не было залипания.
3. **Adaptive recovery после урона:**
   - временное окно стабилизации (снижение скорости врагов, немного более мягкий спавн),
   - чаще pigeon при низком HP,
   - HUD-индикатор активной стабилизации.

## P2 (следующий проход)
1. Тонкая балансировка boss-wave телеграфов (более читаемые атаки/ритм).
2. Яснее объяснить risk/reward двойного прыжка и overheat в туториальном оверлее.
3. Небольшой post-hit knockback tuning против «нечестных» перекрытий.

## P3 (после подтверждения метрик/фидбэка)
1. Простые метрики рантайма (время жизни, причина смерти, где чаще теряют HP).
2. Seeded difficulty presets (casual/normal) без тяжёлых зависимостей.
