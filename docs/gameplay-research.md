# Gameplay research (итерация 1)

## Контекст
Платформер с one-button loop (прыжок/двойной прыжок), mobile-first, цель — выживание и набор очков волнами.

## Быстрые узкие места играбельности

1. **Input-friction на мобиле**
   - На таче есть риск «залипания» Space при отмене touch/уходе пальца с кнопки.
   - Нет буфера прыжка: ранний тап перед приземлением теряется.

2. **Death-spiral после получения урона**
   - После просадки HP волны продолжают идти в прежнем темпе.
   - Лечение (pigeon) может приходить слишком редко именно в момент, когда игроку нужен шанс восстановиться.

3. **Слабая «мягкая адаптация» сложности**
   - Игра даёт i-frames, но этого не всегда хватает на мобильном управлении.
   - Нет короткого recovery-окна для возврата в базовый ритм после ошибки.

## Вывод
Для ощутимого улучшения без ломки core loop нужен P1-пакет: улучшить отзывчивость прыжка + ввести короткий адаптивный recovery-слой после урона.
